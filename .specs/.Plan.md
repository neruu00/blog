# 프로젝트 구조 및 기능 고도화 분석 계획 (.Plan.md)

현재 개발된 블로그 플랫폼(Next.js 15, Supabase, Tailwind 4, Tiptap, NextAuth)의 아키텍처와 코드를 분석하여, 향후 시스템의 안정성, 성능, 유지보수성을 극대화하기 위한 구조적 및 기능적 개선 과제를 정리합니다.

---

## 🏗️ 1. 구조적 최적화 (Architecture & Code Quality)

### 1-1. Server Actions와 React 19 Hooks 연동
- **현황**: `write/page.tsx`, `EditPostClient.tsx` 등 폼 처리 시 `useState(isSubmitting)` 등 클라이언트 상태를 수동으로 제어하며 서버 액션을 호출 중.
- **개선 방안**: Next.js 15 (React 19)의 `useActionState` (또는 `useFormState`) 및 `useFormStatus` 훅을 적극 도입하여 상태 관리 코드(isSubmitting, error, success 로직)를 선언적으로 리팩토링.
- **효과**: 보일러플레이트 코드 대폭 감소 및 동시성 처리 안전성 확보.

### 1-2. TanStack Query와 SSR 캐싱 전략 분리 및 고도화
- **현황**: 서버 컴포넌트에서 Supabase Client를 직접 호출하여 렌더링하고, 수정/삭제 시 `revalidatePath`로 무효화 처리 중. 클라이언트단(TanStack Query)의 명확한 역할 분리가 미흡함.
- **개선 방안**: 
  - 서버 측 데이터 패칭은 `revalidateTag`를 통한 더 세분화된 태그 기반 캐시 무효화(Tag-based Revalidation) 적용.
  - 클라이언트 측 인터랙션(검색, 무한 스크롤, 실시간 필터링)에 대해서만 TanStack Query를 도입하여 명확히 역할 분리.

### 1-3. Supabase RLS (Row Level Security) + NextAuth 연동 강화
- **현황**: 데이터 조작 권한(글 쓰기/삭제)을 Server Actions 첫 줄에서 애플리케이션 레벨의 로직(`requireAdmin`)으로만 차단 중.
- **개선 방안**: NextAuth의 세션 정보를 기반으로 Supabase JWT를 커스텀 발급하거나, Supabase Database 자체의 RLS 정책을 강화. 익명 사용자의 읽기 권한과 관리자(Admin)의 쓰기 권한을 DB 엔진 레벨에서 원천적으로 통제.
- **효과**: 애플리케이션 로직을 우회하는 비정상적인 DB 접근을 원천 차단하여 백엔드 보안성 향상.

## ⚡ 2. 기능적/성능적 최적화 (Performance & UX)

### 2-1. Tiptap 에디터 콘텐츠 렌더링 최적화
- **현황**: 게시글 상세 페이지(`/posts/[id]`)에서 JSON 형태의 콘텐츠를 렌더링할 때 무거운 에디터 런타임 코드가 클라이언트 번들에 포함될 우려가 있음.
- **개선 방안**: 
  - 작성/수정 완료 시 백엔드 로직(`post.ts` Server Action)에서 `@tiptap/html`의 `generateHTML` 함수를 통해 Static HTML 문자열을 사전 생성하여 별도 DB 컬럼에 저장.
  - 읽기 전용 페이지에서는 에디터 라이브러리 없이 단순 `dangerouslySetInnerHTML` 또는 파싱 라이브러리만 사용하여 가볍게 렌더링.
- **효과**: 초기 로드 속도 단축, 클라이언트 JS 번들 크기 대폭 감소, SEO(검색 엔진 최적화) 성능 극대화.

### 2-2. 이미지 최적화 및 CDN 파이프라인
- **현황**: Tiptap을 통한 이미지 업로드 구현이 되어 있으나 최적화 파이프라인 부재 여부 확인 필요.
- **개선 방안**: Supabase Storage를 활용할 때 이미지 포맷을 WebP로 변환하고, 렌더링 시 Next.js의 `next/image`를 강제하여 레이지 로딩과 반응형 리사이징(srcSet)을 완벽 지원하도록 파이프라인 구축.

### 2-3. 동적 Open Graph (OG) 이미지 및 SEO 메타데이터 자동화
- **현황**: 기본적인 정적 메타데이터 적용 수준.
- **개선 방안**: `next/og` 모듈을 연동하여 각 게시글의 제목, 태그, 배경을 조합한 동적 OG 썸네일(`/api/og?title=...`) 자동 생성 체계 구축. 동적 `generateMetadata`로 SNS 공유 시 노출력 강화.

## 🎯 3. 유지보수 및 확장성 (Maintainability)

### 3-1. 공통 UI 컴포넌트 시스템 구축 완료 (디자인 시스템화)
- **현황**: `Modal`, `ToastContainer`가 분리되었으나, 여전히 많은 버튼, 인풋 등이 인라인 스타일을 가짐.
- **개선 방안**: CVA (Class Variance Authority) 및 `tailwind-merge`를 도입하여 `Button`, `Input`, `Badge` 등 원시 컴포넌트를 `components/ui/`에 완벽히 분리하고 디자인 시스템 토대 완성.

### 3-2. 포트폴리오 데이터 소스 분리 (CMS화)
- **현황**: 포트폴리오 프로젝트, 기술 스택 등이 React 컴포넌트 내부에 하드코딩.
- **개선 방안**: 포트폴리오 데이터를 Supabase 테이블로 이동시키고, 어드민 페이지에서 이력을 관리할 수 있는 CMS 기능을 연동하여 코드 배포 없이 이력서를 수정할 수 있도록 확장.
