# 🏆 Golden Dataset (Evaluation Scenarios)

이 문서는 AI 에이전트의 성능을 평가하기 위한 새로운 핵심 시나리오와 통과 기준을 정의합니다.

## 시나리오 1: 게시글 페이지네이션 (Pagination)
- **목표**: 게시글 목록이 많아질 경우를 대비하여 서버 사이드 페이지네이션을 구현한다.
- **통과 기준**:
    - [ ] 한 페이지에 10개(또는 설정된 수)의 게시글만 표시됨
    - [ ] 이전/다음 버튼 및 페이지 번호 UI가 정상 작동함
    - [ ] URL 쿼리 파라미터(`?page=n`)와 연동되어 새로고침 후에도 상태가 유지됨

## 시나리오 2: 작성/수정 편의성 개선 (임시저장 및 뒤로가기)
- **목표**: 게시글 작성 중 데이터를 잃지 않도록 임시저장 기능을 추가하고, 의도치 않은 이탈을 방지한다.
- **통과 기준**:
    - [ ] 로컬 스토리지 또는 DB를 활용한 '임시저장' 버튼 및 자동 저장 로직 구현
    - [ ] 변경 사항이 있을 때 '뒤로가기' 클릭 시 확인 모달(브라우저 confirm 등) 표시
    - [ ] '취소' 또는 '뒤로가기' 버튼이 작성/수정 폼에 적절히 배치됨

## 시나리오 3: 인터랙티브 게임 페이지 (Game Console Experience)
- **목표**: iframe을 활용하여 배포된 게임을 불러오고, 닌텐도 스위치 스타일의 화려한 카트리지 삽입 애니메이션을 구현한다.
- **통과 기준**:
    - [ ] **카트리지 인터랙션**: 마우스 위치에 따른 3D 회전(Rotate) 및 반짝이는 오버레이 효과
    - [ ] **카트리지 선택**: 클릭 시 중앙으로 이동 및 확대되며 게임 설명 표시
    - [ ] **게임기 디자인**: 뷰포트 하단에 컨트롤러가 달린 스위치 형태의 게임기 본체가 살짝 노출됨
    - [ ] **삽입 애니메이션**: '게임하기' 클릭 시 카트리지가 아래로 이동하며 게임기에 삽입되는 연출
    - [ ] **게임 실행**: 삽입 완료 후 게임기가 중앙으로 올라오며 iframe 콘텐츠 로드
    - [ ] **성능**: 복잡한 애니메이션 중에도 프레임 드랍이 없으며 모바일 반응형 고려

---

## 공통 평가 항목 (Score Card)
1. **디자인 완성도 (40%)**: 고급스러운 애니메이션, 인터랙션의 부드러움, 프리미엄 UI
2. **코드 품질 (30%)**: Framer Motion 등 애니메이션 라이브러리의 효율적 사용, 컴포넌트 구조화
3. **사용자 경험 (30%)**: 로딩 상태 처리, 명확한 피드백, 직관적인 네비게이션
