# 🏆 Golden Dataset (Evaluation Scenarios)

이 문서는 AI 에이전트의 성능을 평가하기 위한 핵심 시나리오와 통과 기준을 정의합니다.

## 시나리오 1: 이미지 업로드/로딩 스켈레톤 구현
- **목표**: 에디터에서 이미지를 삽입하거나 불러올 때, 이미지가 로딩(pending) 상태이면 해당 공간에 placeholder skeleton을 표시한다.
- **통과 기준**:
    - [ ] 이미지 업로드 중 또는 초기 로딩 시 `Skeleton` 컴포넌트가 올바르게 표시됨
    - [ ] 이미지 로딩 완료 후 스켈레톤에서 실제 이미지로 부드럽게 전환됨
    - [ ] 업로드 실패 시 에러 상태 UI 처리 및 스켈레톤 제거
    - [ ] Tailwind 4의 `animate-pulse` 등을 사용하여 디자인 시스템의 스켈레톤 스타일 준수

## 시나리오 2: 좋아요 버튼 낙관적 업데이트 버그 수정
- **목표**: 좋아요 클릭 시 UI가 즉시 반응하지 않는 현상을 수정한다.
- **통과 기준**:
    - [ ] `useOptimistic` 훅 또는 TanStack Query의 `onMutate` 사용
    - [ ] 서버 응답 지연(2초) 시에도 UI는 즉시 숫자 변경 및 하이라이트 표시
    - [ ] 서버 에러 발생 시 원래 상태로 롤백 로직 포함

## 시나리오 3: 서버 액션 보안 강화
- **목표**: 게시글 작성 액션에서 Admin 권한이 없는 사용자의 접근을 차단한다.
- **통과 기준**:
    - [ ] `auth.action.ts` 또는 유사 파일의 `getServerSession`을 통한 권한 체크 로직 추가
    - [ ] 관리자가 아닐 경우 명확한 에러 메시지 반환 또는 Redirect
    - [ ] Zod를 이용한 Input 데이터 검증(Schema validation) 수행

---

## 공통 평가 항목 (Score Card)
1. **코드 품질 (40%)**: 타입 정의의 정확성, 중복 코드 최소화, 가독성
2. **명세 준수 (30%)**: `.specs/` 및 `.agent/rules/` 지침 준수 여부
3. **작동 완결성 (30%)**: 에러 처리, 에지 케이스 고려, UI 반응성
